# EW PRO · 문의 폼 + DB 연동 설정 가이드

이 사이트의 **Contact 문의 폼**을 Supabase(무료 데이터베이스)에 연결하는 방법입니다.
설정 전에도 폼은 **자동으로 이메일 전송 방식**으로 동작하므로, 급하지 않으면 나중에 해도 됩니다.

소요 시간: 약 10분 · 비용: 무료

---

## 준비물
- `site/` 폴더 (지금 배포 중인 파일들)
- 그 안의 `config.js`, `supabase-setup.sql` (이 패키지에 포함됨)

---

## STEP 1 — Supabase 프로젝트 만들기
1. https://supabase.com 접속 → **Start your project** → GitHub 계정으로 로그인
2. **New project** 클릭
   - Name: `ewpro`
   - Database Password: 아무 강한 비밀번호 (메모해두기)
   - Region: **Northeast Asia (Seoul)** 선택
3. **Create new project** → 1~2분 기다리면 생성 완료

## STEP 2 — 데이터베이스 테이블 만들기
1. 왼쪽 메뉴 **SQL Editor** 클릭 → **New query**
2. 이 패키지의 **`supabase-setup.sql`** 파일 내용을 전부 복사해 붙여넣기
3. 오른쪽 아래 **RUN** 클릭 → "Success" 나오면 완료
   - 왼쪽 **Table Editor**에 `inquiries` 테이블이 생겼는지 확인

## STEP 3 — 연결 키 복사해서 넣기
1. 왼쪽 아래 **⚙ Project Settings → API** 메뉴
2. 두 가지를 복사:
   - **Project URL** (예: `https://abcdxyz.supabase.co`)
   - **Project API keys → `anon` `public`** (긴 문자열 `eyJ...`)
3. `config.js` 파일을 열어 두 값을 붙여넣기:
   ```js
   window.EWPRO_SUPABASE = {
     url:     "https://abcdxyz.supabase.co",   // ← Project URL
     anonKey: "eyJhbGciOi..."                  // ← anon public key
   };
   ```
   > `anon public` 키는 공개되어도 안전한 키입니다. DB 보안은 SQL에서 설정한 RLS 정책이 담당하므로, 방문자는 문의를 "제출"만 할 수 있고 다른 사람 문의를 볼 수는 없습니다.

## STEP 4 — GitHub에 올려서 배포
- 수정한 **`config.js`** (그리고 이 패키지의 나머지 파일)를 GitHub 저장소에 업로드 → Commit
- Vercel이 자동 재배포 → 끝!

## 접수된 문의 확인하는 법
당분간은 **Supabase 대시보드**에서 바로 봅니다:
- Supabase → **Table Editor → inquiries** 테이블
- 방문자가 남긴 문의가 행으로 쌓입니다 (회사명·담당자·연락처·내용·시각)
- `status` 값을 `new → replied → done` 으로 바꿔가며 관리

> 사이트 안에 **관리자 로그인 + 문의 목록 대시보드**가 필요하면, `supabase-setup.sql` 하단의 "관리자 조회 정책"을 켠 뒤 별도 admin 페이지를 만들면 됩니다. 원하시면 만들어 드릴게요.

---

## (선택) 신규 문의 알림 받기
문의가 들어올 때 이메일/슬랙 알림을 받고 싶다면:
- Supabase **Database → Webhooks** 에서 `inquiries` INSERT 시 웹훅 발송
- 또는 **Edge Functions**로 Resend(이메일)·Slack 연동

이 부분까지 필요하면 알려주세요.

## 동작 요약
| 상황 | 폼 동작 |
|---|---|
| `config.js` 미설정 | 방문자의 메일앱으로 문의 내용이 채워진 이메일 열림 (안전 폴백) |
| `config.js` 설정 완료 | Supabase `inquiries` 테이블에 바로 저장 + "접수 완료" 화면 |
| 저장 중 오류 | 이메일 폴백으로 자동 전환 (문의 유실 방지) |
